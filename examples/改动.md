# Tensor Compiler 架构重构改动总结

## 一、概述

本次重构引入了分层编译思想，实现了三大核心改进：
1. **CodeGen 分层架构**：Graph IR → TIR → C++ AST → C++ Source
2. **FFI 安全性改进**：pybind11 + DLPack 零拷贝 + 异常边界
3. **控制流与符号推导**：AST/Bytecode 解析 + SymPy 符号引擎

---

## 二、CodeGen 分层编译架构

### 2.1 文件结构

```

├── ir/
│   ├── __init__.py          # IR 模块入口
```

### 2.2 C++ AST 节点体系 (`ir/cpp_ast.py`)

**核心节点类型**：

| 类别 | 节点类型 | 说明 |
|------|----------|------|
| 表达式 | `Identifier`, `Literal`, `BinaryOp`, `UnaryOp`, `TernaryOp` | 基础表达式 |
| 表达式 | `Call`, `Index`, `Cast` | 函数调用、数组索引、类型转换 |
| 语句 | `Assign`, `VarDecl`, `Block`, `ForLoop`, `WhileLoop`, `If` | 控制流语句 |
| 高级 | `FunctionDecl`, `StructDecl`, `Program` | 函数、结构体、程序 |

**示例**：
```python

loop = ForLoop(
    init=VarDecl(var_type="int", name="i", init=Literal(0)),
    cond=BinaryOp(Identifier("i"), "<", Literal(10)),
    update=Assign(Identifier("i"), BinaryOp(Identifier("i"), "+", Literal(1))),
    body=Block(stmts=[...])
)
code = generate_cpp(loop)
```

### 3.1 pybind11/nanobind 绑定 (`runtime_pybind.py`)

**核心类**：

- `PybindJITModule`: 使用 pybind11 的 JIT 模块

**C++ 绑定代码**：

```cpp
class Tensor {
public:

    void* data();
    int64_t numel();

PYBIND11_MODULE(tensor_cpu_native, m) {
    py::class_<Tensor>(m, "Tensor")
        .def(py::init<py::array_t<float>>())
        .def("data", &Tensor::data)
        .def("numel", &Tensor::numel);
    
    py::class_<CompiledKernel>(m, "CompiledKernel")
        .def(py::init<std::string, std::string>())
        .def("run", &CompiledKernel::run);
    
    py::register_exception_translator([](std::exception_ptr p) {
        try { if (p) std::rethrow_exception(p); }
        catch (const KernelError& e) { PyErr_SetString(PyExc_RuntimeError, e.what()); }
    });

```

### 3.2 DLPack 零拷贝机制 (`runtime_pybind.py`)

**核心类**：
```python
class DLPackTensor:
    """DLPack-based zero-copy tensor wrapper."""
    
    def __init__(self, array: np.ndarray):
        self._array = array
    
    def __dlpack__(self, stream: Optional[int] = None) -> Any:
        """Return a DLPack capsule for zero-copy sharing."""
        return self._array.__dlpack__()
    
    def __dlpack_device__(self) -> Tuple[int, int]:
        """Return (device_type, device_id)."""
        return (1, 0)  # kDLCPU
    
    @staticmethod
    def from_dlpack(capsule: Any) -> np.ndarray:
        """Create numpy array from DLPack capsule."""
        return np.from_dlpack(capsule)
```

**优势**：
- 跨框架张量共享（PyTorch, JAX, NumPy 均支持）
- 直接复用内存，消灭 FFI 传参时的拷贝
- 消除裸指针强转风险

### 3.3 跨语言异常边界

**C++ 侧**：
```cpp
class KernelError : public std::runtime_error {
public:
    explicit KernelError(const std::string& msg, int code)
        : std::runtime_error(msg), error_code(code) {}
    int error_code;
};
```

**Python 侧**：
```python
def register_exception_translations(py::module& m) {
    static py::exception<KernelError> exc(m, "KernelError");
    py::register_exception_translator([](std::exception_ptr p) {
        try { if (p) std::rethrow_exception(p); }
        catch (const KernelError& e) { exc(e.what()); }
    });
}
```

**收益**：
- 从 "Segmentation Fault 导致进程崩溃" 优化为
- "抛出带有清晰 C++ 堆栈信息的 Python 异常"

### 4.3 SymPy 符号推导引擎 (`symbolic.py`)

**核心类**：
```python
@dataclass
class SymbolicDim:
    """A symbolic dimension expression."""
    expr: Any  # SymPy expression
    
    def __add__(self, other) -> "SymbolicDim": ...
    def __mul__(self, other) -> "SymbolicDim": ...
    def simplify(self) -> "SymbolicDim": ...
    def evaluate(self, bindings: Dict[str, int]) -> int: ...

class SymbolicEngine:
    """Symbolic shape inference and constraint solving engine."""
    
    def get_symbol(self, name: str) -> Symbol:
        """Get or create a symbolic variable."""
    
    def add_equality(self, dim1: SymbolicDim, dim2: SymbolicDim):
        """Add equality constraint: dim1 == dim2."""
    
    def solve_constraints(self) -> Optional[Dict[str, int]]:
        """Solve all constraints."""
    
    def are_equal(self, dim1: SymbolicDim, dim2: SymbolicDim) -> bool:
        """Check if two dimensions are provably equal."""

class SymbolicShapeInference:
    """Symbolic shape inference for tensor operations."""
    
    def infer_elementwise_shape(self, lhs, rhs) -> SymbolicShape:
        """Infer shape for elementwise ops with broadcasting."""
    
    def infer_matmul_shape(self, a, b) -> SymbolicShape:
        """Infer shape for matmul, adding K equality constraint."""
    
    def infer_reduce_shape(self, input, axes, keepdims) -> SymbolicShape:
        """Infer shape for reduction operations."""
```

## 五、架构对比

### 原有架构
```
Graph IR ──字符串拼接──> C++ Source ──ctypes──> Python
```

### 新架构
```
Graph IR ──Lowering──> TIR ──Conversion──> C++ AST ──Codegen──> C++ Source
    │                │                    │
    │                └── 优化 Pass ───────┘
    │
    └── SymPy 符号推导 ──> 编译期验证
    │
    └── AST/Bytecode 解析 ──> 控制流捕获
    │
    └── pybind11 + DLPack ──> 零拷贝 FFI
```
