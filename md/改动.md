这次主要改了 6 块，核心是“修 4 个问题 + 补 2 类回归”。

1. 修复 `MAX` 的图模式反向错误  
- 文件: vjp.py  
- 改动: `OpType.MAX` 不再是 `pass`，改为构建 `max` mask 并乘回上游梯度；同时修了 `keepdims=False` 时先扩维再广播的逻辑。

2. 为 `MAX` 反向引入内部 `EQ` 算子支持  
- 文件: ops.py  
- 文件: shape_inference.py  
- 文件: codegen.py  
- 改动: 新增 `OpType.EQ`，接入 shape 推导与 codegen；比较实现用了带容差的浮点比较。

3. 修复 fused symbolic shape / fused matmul codegen  
- 文件: codegen.py  
- 改动:  
  - `FUSED_MATMUL_BIAS*` 增加 symbolic shape 传播。  
  - `_emit_fused_matmul` 改为使用符号维度，不再依赖 trace 时的 literal shape。  
  - fused 循环索引改成 `long long`。  
  - 额外增强 `BROADCAST_TO` 的 expand-dims 符号推导（比如 `(B,) -> (B,1)`）。

4. 修复 `LazyTensor.softmax` 参数不匹配  
- 文件: lazy.py  
- 改动: 不再把 `eps` 传给 `Tensor.softmax`（避免 `TypeError`）。

5. 去掉 pass 静默失败  
- 文件: runtime.py  
- 改动: 删除 `compile_graph` 中吞异常的 `except Exception: pass`，现在优化 pass 出错会直接暴露。

6. 新增两类回归测试  
- 文件: test_training.py  
- 新增: `test_graph_reduce_gradients_numerical`（`mean/max` 图模式梯度 vs 数值梯度）  
- 文件: test_jit_core.py  
- 新增: `test_fused_symbolic_shapes_dynamic_batch`（fusion 后同 rank 不同 batch 的动态形状一致性）

已验证结果：这两个新增回归都跑通过了。
